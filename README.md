## Варіант завдання

* **Завдання:** Модифікувати лабораторну роботу №3 наступним чином: для літер, слів, речень, розділових знаків та тексту створити окремі класи. Слово повинно складатися з масиву літер, речення з масиву слів та розділових знаків, текст з масиву речень. Замінити послідовність табуляцій та пробілів одним пробілом.
* **Дія з Лаб №3:** Створити масив об'єктів (в даному випадку — слів), виконати сортування та пошук конкретного об'єкта за логікою `equals()`.

## Опис реалізації

**1. Інтерфейс `SentenceElement`**
* Об'єднує складові частини речення (слова та розділові знаки) для зберігання в одному масиві.

**2. Клас `Letter`**
* Представляє окрему літеру тексту.
* Містить символ типу `char` та методи для його отримання.

**3. Клас `Punctuation`**
* Представляє розділовий знак.
* Реалізує інтерфейс `SentenceElement`.

**4. Клас `Word`**
* Представляє окреме слово.
* Складається з масиву об'єктів `Letter`.
* Перевизначені методи `equals()`, `toString()`, а також реалізовано метод `getLength()` для отримання довжини слова.

**5. Клас `Sentence`**
* Представляє речення, яке складається з масиву `SentenceElement` (слів та розділових знаків).
* Містить логіку для коректного збирання речення з пробілами між словами.

**6. Клас `Text`**
* Головний клас для обробки тексту. Складається з масиву об'єктів `Sentence`.
* У конструкторі реалізовано парсинг: очищення тексту від зайвих пробілів та табуляцій (заміна на один пробіл) та розбиття рядка на речення, слова та пунктуацію.

**7. Клас `Main`**
* Демонструє роботу парсера (очищення тексту).
* Створює масив об'єктів `Word`.
* Виконує сортування та пошук елементів у масиві.

## Сортування та пошук

* **Сортування колекції слів:**
  * За довжиною слова (спадання) за допомогою `Comparator.comparing().reversed()`.
  * За довжиною слова (зростання) за допомогою `Comparator.comparing()`.
* **Пошук:** Знаходження конкретного слова у масиві за допомогою циклу та перевизначеного методу `equals()`.
